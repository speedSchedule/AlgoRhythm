# SpeedSchedule 프로젝트 개요

## 문제 (Problem)
**현실:** 기존 학사 시스템은 자동화된 것처럼 보이나, 실제로는 관리자가 엑셀을 이용해 수동으로 수정해야 하는 ‘반자동’ 구조임.
**Pain Point:**
낙후된 UI로 인해 시간표 확인, 수정, 보결 및 교환 요청 과정에서 업무 처리 효율이 매우 저하됨.
고정된 로직(가나다순, 순번제 등)에 의존하여 교사별 선호도, 숙련도, 업무 부담 등 개별 조건을 반영하는 데 한계가 있음.
특정 인원에게 업무가 집중되는 불균형으로 인해 행정 신뢰도 및 교사 만족도가 하락함.

## 해결 아이디어 (Solution)
**핵심:** AI 기반 인력 운영 최적화 플랫폼, 'SpeedSchedule'.
**주요 기능:**
자연어 기반 요구사항 입력(LLM 파싱)을 통해 복잡한 제약조건을 자동으로 설정.
하드 제약조건을 보장하는 알고리즘으로 시간표 자동 생성 및 복수의 최적화 대안 제공.
결원, 보결, 교환 요청 등 실시간 변수를 통합 관리하는 라이브 학사 관리.

## 기술/구현 (Tech Stack & Implementation)
**Frontend:** React + Vite 기반의 인터랙티브 캘린더 UI (Vercel 배포)
**Backend:** Spring Boot + JPA 기반 REST API, MySQL(인력/스케줄 데이터), Redis(실시간 상태 캐싱)
**AI/Optimization:** 하드 제약조건 보장 알고리즘 구현
**OpenAI/Claude API:** 자연어 기반 제약조건 추출 및 결원 발생 시 최적 매칭 추천
**Infra:** AWS EC2 + RDS, Docker 및 GitHub Actions를 활용한 CI/CD 파이프라인

## MVP 및 배포 전략 (MVP & Deployment)
**MVP 기능:** 시간표 자동 생성, 캘린더 기반 수정, 보결·교환 요청 관리, 관리자 승인 프로세스, 자연어 기반 요구사항 인식.
**배포 방식:**
- 백엔드: AWS EC2 + Docker 컨테이너 배포
- 프론트엔드: Vercel을 통한 서빙
유지보수 효율을 위한 이중 배포 환경 구축

## 차별성 (Differentiation)
**기존 제품과의 차이:** 컴시간알리미, 스쿨마스터 등 기존 도구가 관리자의 '수동 데이터 입력'에 의존하는 정적인 설치형 도구라면, SpeedSchedule은 AI가 교사의 의도를 자연어로 직접 파악하는 '지능형 비서'형 플랫폼임.
**독특성:** 하드 제약조건을 완벽히 보장하는 안정적인 알고리즘과 AI 기반의 라이브 학사 관리 기능을 결합하여, 실시간으로 발생하는 돌발 상황에 유연하게 대응함.
**우수성:** 단순 시간표 생성을 넘어 교사의 업무 환경 만족도를 실질적으로 높이고, 웹 기반 플랫폼을 통해 행정의 투명성과 공정성을 확보함.

## PMF(Product-Market Fit) 근거 (산학트랙 필수)
**시장성:** 학교 현장의 낡은 시스템은 현대적 UI와 유연한 로직에 대한 갈증이 매우 큼.
**효율성:** 단순 반복 업무인 시간표 편성 과정을 자동화하여 관리자의 행정 노동력을 획기적으로 절감함.
**확장성:** 고교학점제 등 점차 복잡해지는 학사 운영 환경에서 가변적 요구사항을 즉각적으로 처리할 수 있는 유일한 대안이 될 것임.

# 고객 인터뷰 및 검증 과정

## 인터뷰 과정
**대상:** 교무부장 A, 일반 교사 B

**방법:** 카카오톡 오픈채팅을 통해 기존 시스템 사용 시 발생하는 가장 빈번한 문제 상황(시간표 편성 및 실시간 변경)에 대해 질문하고, 우리의 해결 아이디어(자연어 설정, 실시간 AI 매칭)를 제시했을 때의 반응을 수집함.

## 인터뷰 결과
**교무부장 A의 반응:** 기존 시스템은 입력 단계부터가 수작업에 가깝다고 볼 수 있음. 더군다나 고교학점제 때문에 조건이 복잡해졌는데 이걸 AI가 알아듣고 자동으로 제약조건으로 바꿔준다면, 시간표 작성에 걸리는 시간을 50% 이상 줄일 수 있을 것 같음.

**교사 B의 반응:** 수업 변경이나 보결 요청할 때 단톡방에 공지하고 눈치 보는 게 너무 비효율적임. 시스템에서 실시간으로 누가 시간 되는지 매칭해주고 관리자가 승인만 하면 바로 반영되는 방식이면 교사 만족도가 훨씬 올라갈 것 같음. 

## 차별성 및 우수성에 대한 논리적 근거
**독특성:** 단순히 엑셀을 디지털화한 기존 제품들과 달리, LLM 기반 자연어 파싱을 통해 관리자의 데이터 입력 노동을 제거함.

**우수성:** 하드 제약조건을 준수하는 알고리즘과 AI 대리 근무자 매칭을 통해, 기존 시스템에서 불가능했던 실시간 학사 라이브 대응이 가능함.

-> 인터뷰 결과, 관리자의 행정 효율성과 실무 교사의 정보 접근성이라는 양측의 니즈를 동시에 만족시킬 수 있는 PMF(Product-Market Fit)를 확인함.
